## 2. 问题清单（按严重程度）

### P0 — 必须立即修复（阻塞编译 / 集成）

#### P0-1：csproj 未引用平台 DLL（编译失败）

**现象**（MSBuild 实测）：

```
error CS0246: 未能找到类型或命名空间名"InterfaceDefine"
error CS0246: 未能找到类型或命名空间名"MainModule"
error CS0246: 未能找到类型或命名空间名"ProcessModuleBase"
error CS0246: 未能找到类型或命名空间名"ModuleAlarmEventArgs"
error CS0246: 未能找到类型或命名空间名"TaskItemSetting"
```

**原因**：源码已 `using InterfaceDefine` / `using MainModule`，临时替代 `.cs` 已删除，但 `ProcessModules.csproj` **没有** `<Reference Include="InterfaceDefine">` / `MainModule`。

**影响**：整库无法生成 `ProcessModules.dll`，平台无法集成。

**修复**（见《操作指南.md》第四节）：

```xml
<Reference Include="InterfaceDefine">
  <HintPath>InterfaceDefine.dll</HintPath>
  <Private>True</Private>
</Reference>
<Reference Include="MainModule">
  <HintPath>MainModule.dll</HintPath>
  <Private>True</Private>
</Reference>
```

将 DLL 放到 HintPath 对应路径（可从现场平台或 `DOMOPlatform` 获取），再 `Rebuild`。

### P1 — 高优先级（运行时正确性 / 安全）

#### P1-1：硬件位置回调可能跨线程更新 UI

**位置**：

- `XyzControllerHub.Service_PositionUpdated` → `ApplyPosition` → `AxisController.UpdateCurrent` → `Changed`
- 各 `RunForm.Hub_Changed` → `SyncUiFromHub()` 直接改控件

**风险**：真实控制卡回调常在非 UI 线程 → `InvalidOperationException`（跨线程访问控件）。

**建议**：

```csharp
// RunForm.Hub_Changed 内
if (InvokeRequired)
{
    BeginInvoke(new EventHandler(Hub_Changed), sender, e);
    return;
}
SyncUiFromHub();
```

四个窗体均需处理：`MainControl\RunForm`、`UnifiedRunForm`、`PointJump\RunForm`、`Trajectory\RunForm`。

---

#### P1-2：`Hub.Changed` 只订不退，窗体重开会重复刷新

**位置**：各 `RunForm.OnLoad`：

```csharp
_hub.Changed += new EventHandler(Hub_Changed);
// 无 -= ，无 FormClosed 清理
```

**风险**：`ShowRunForm` 多次创建/嵌入时，同一 Hub 上堆积多个 handler → 重复 `SyncUiFromHub`、已释放控件仍被调用。

**建议**：在 `OnFormClosed` / `Dispose` 中：

```csharp
if (_hub != null)
    _hub.Changed -= Hub_Changed;
```

---

#### P1-3：`ApplyPosition` 用后端 Target 覆盖本地目标

**位置**：`XyzControllerHub.ApplyPosition`：

```csharp
if (pos.Target != null)
{
    for (...)
        _axes[i].SetTarget(pos.Target[i]);  // 覆盖用户/Action 设定的 Target
}
```

**风险**：若硬件反馈包里带了 Target（或骨架误填），会冲掉用户正在设定的目标，造成「拖不动 / 跳回」。

**建议**：

- 生产约定：反馈只填 `Actual`（及状态位），`Target` 置 `null`；或  
- Hub 侧**忽略**反馈中的 `Target`，仅更新 `Current`。

---

#### P1-4：`StopAll` / `ESTOP` 未调用 Hub 级 `Stop`/`EmergencyStop`

**位置**：`MainControlProcessModule.StopAll`：

```csharp
foreach (AxisJogService s in _jogServices)
    s.EmergencyStop();  // 仅把 Target 拉回 Current
// 未调用 _hub.EmergencyStop() / _hub.Stop()
```

**风险**：JOG 服务只改本地 Target；**已下发到硬件的运动不会被急停指令打断**。

**建议**：

```csharp
public override bool StopAll()
{
    foreach (AxisJogService s in _jogServices)
        s.EmergencyStop();
    if (_hub != null)
        _hub.EmergencyStop();  // 或 Stop()，按安全策略
    // ...
}
```

PointJump / Trajectory 的 `STOP` 同理，应走 `_hub.Stop()`。

### P2 — 中优先级（兼容性 / 可维护性）

#### P2-1：C# 7 内联 `out` 声明（工控机旧编译器风险）

**位置**：

```csharp
// MainControlProcessModule / PointJumpProcessModule
float.TryParse(..., out float x);
```

**说明**：.NET 4.6.1 + 现代 VS 可编过；若现场仍用极旧 C# 编译器，会失败。

**建议**（与项目「避免 Lambda」策略一致）：

```csharp
float x;
if (!float.TryParse(param[1].ToString(), out x))
```

---

#### P2-2：`InitAll` 部分失败仍标记 `_initialized`

```csharp
ok = RegisterAndInit(...) && ok;
// ...
_initialized = ok;  // 若第二个失败，第一个已注册
return ok;
```

**风险**：返回 `false` 后再次 `InitAll` 因 `_initialized==false` 会**重复 Register** 覆盖字典，但已失败模组状态不清晰。

**建议**：全部成功才 `_initialized = true`；失败时记录哪个模组失败，或允许重试未成功项。

---

#### P2-3：Setting.Save 无 try/catch，磁盘失败直接抛异常

Global/Project 的 `Save()` 直接 `XMLSerializationHelper.SaveToFile`，无捕获。

**建议**：与 Load 对称，catch 后返回 `false` 并写 Debug/报警。

---

#### P2-4：U 轴有模型无完整 UI/Action

Hub 与 GlobalSetting 含 U 轴；MainControl JOG/Action 仅 X/Y/Z。属功能缺口，非 bug，但文档应标明「U 轴预留」。

---

#### P2-5：`PlatformMotionService` 为骨架

`ExecuteCommand` 接受指令但不驱动硬件；`GetPosition` 返回缓存。符合「骨架」定位，**上线前必须替换真实实现**（可用 `DOMOPlatform\SimulatedMotionService.cs` 作调试参考，生产禁用模拟）。

## 3. 做得好的地方

1. **分层清晰**：RunForm 只绑 UI；Hub 打包 `MotionCommand`；`IMotionService` 隔离硬件。  
2. **工控机友好**：事件用命名方法 + `new EventHandler(...)`，避免 Lambda。  
3. **平台入口完整**：`InitAll` / `InjectServiceToAll` / `SubscribeAlarms` / `StopAll` / `SaveAll` / `CloseAll`。  
4. **DOMO 对齐**：`Init` / `ShowRunForm` / `Action` / `SetParam` / `GetParam` / 项目事件订阅结构正确。  
5. **位置铁律**：接口与注释明确禁止伪造 `Actual`；骨架不编造轨迹。  
6. **设计器安全**：业务初始化放在 `OnLoad`，避免 VS 设计器红屏。  
7. **文档体系**：操作指南 / 接口文档 / 启动指南已同步，利于集成。

## 4. 架构与数据流（审查确认）

```
平台 ProcessModuleManager
  → *ProcessModule (ProcessModuleBase)
    → XyzControllerHub
      → IMotionService.ExecuteCommand(MotionCommand)
      ← PositionUpdated / GetPosition(AxisPosition)
    → RunForm (Hub.Changed → SyncUi)
```

**指令路径**：UI / Action → Hub → MotionCommand → 服务 → 硬件  
**反馈路径**：硬件 → PositionUpdated → Hub.Current → Changed → UI
