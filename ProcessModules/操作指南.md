# ProcessModules 工艺模组操作指南（新手版）

---

## 一、这个项目是什么？

ProcessModules 是一个**完全独立的工艺模组类库**，用于控制 XYZU 四轴运动。
它可以单独编译成一个 `ProcessModules.dll`，不依赖任何其他项目。

**文件夹位置**：`d:\zm\C#\XyzController\ProcessModules\`

---

## 二、怎么打开和编译？

### 方法 1：用 Visual Studio 打开

1. 双击打开 `ProcessModules\ProcessModules.sln`
2. 按 `Ctrl + Shift + B` 编译
3. 看到输出窗口显示 `ProcessModules -> ...\ProcessModules.dll` 就成功了

### 方法 2：用命令行编译

```powershell
cd d:\zm\C#\XyzController\ProcessModules
& "D:\Program Files\Microsoft Visual Studio\18\Community\MSBuild\Current\Bin\MSBuild.exe" ProcessModules.sln /p:Configuration=Debug
```

## 四、明天引入真实 DLL 的操作步骤

### 第 1 步：放入 DLL

把 `InterfaceDefine.dll` 和 `MainModule.dll` 复制到 `ProcessModules\` 文件夹下。

### 第 2 步：删除 6 个临时文件

删除以下文件（它们是 DLL 的临时替代品）：

```
ProcessModules\ProcessModuleBase.cs
ProcessModules\ProcessModuleEnvironment.cs
ProcessModules\XmlSerializationHelper.cs
ProcessModules\ModuleVariables.cs
ProcessModules\MachineStatus.cs
ProcessModules\ModuleAlarmEventArgs.cs
```

### 第 3 步：修改 csproj

用记事本打开 `ProcessModules.csproj`：

**删除这 6 行**（搜索文件名就能找到）：
```xml
<Compile Include="MachineStatus.cs" />
<Compile Include="ModuleAlarmEventArgs.cs" />
<Compile Include="ModuleVariables.cs" />
<Compile Include="ProcessModuleBase.cs" />
<Compile Include="ProcessModuleEnvironment.cs" />
<Compile Include="XmlSerializationHelper.cs" />
```

**在 `<ItemGroup>` 的 Reference 区域添加**：
```xml
<Reference Include="InterfaceDefine">
  <HintPath>InterfaceDefine.dll</HintPath>
</Reference>
<Reference Include="MainModule">
  <HintPath>MainModule.dll</HintPath>
</Reference>
```

### 第 4 步：重新编译

`Ctrl + Shift + B`，成功即可。

> 注意：代码中的 `using MainModule;` 和 `using InterfaceDefine;` 不需要改，
> 因为真实 DLL 里的命名空间和我们的临时文件是一样的。

## 五、怎么对接真实硬件？

### 核心概念

```
你写的硬件驱动代码 → 实现 IMotionService 接口 → 注入到模组 → 前端自动显示真实位置
```

### 操作步骤

1. 打开 `Logic\PlatformMotionService.cs`（这是模板）
2. 在 `ExecuteCommand` 方法里填入你的硬件调用代码
3. 在硬件位置回调中调用 `RaisePositionUpdated(位置)` 推送真实位置
4. 在平台启动代码中：

```csharp
// 创建你的服务
var svc = new PlatformMotionService();
svc.Open();

// 初始化模组
ProcessModuleManager.InitAll();

// 一行代码注入（所有模组立即获得运动控制能力）
ProcessModuleManager.InjectServiceToAll(svc);
```

### 绝对不能做的事

> ❌ **严禁伪造位置数据！**
> `Actual`（实际位置）必须来自真实硬件反馈。
> 如果硬件没连，位置就是 0，不要编造数据让界面"看起来在动"。

## 六、怎么添加新的指令？

比如要加一个"设置加速度"功能：

### 第 1 步：加指令类型

打开 `Logic\MotionCommand.cs`，在枚举最后加一行：

```csharp
public enum MotionCommandType
{
    // ... 现有的 ...
    SetAcceleration,   // ← 新加的
}
```

### 第 2 步：在 Hub 加便捷方法

打开 `Logic\XyzControllerHub.cs`，加一个方法：

```csharp
public void SetAcceleration(float acc)
{
    MotionCommand cmd = new MotionCommand(MotionCommandType.SetAcceleration);
    cmd.Speed = acc;
    SendCommand(cmd);
}
```

### 第 3 步：在后端实现中处理

打开 `Logic\PlatformMotionService.cs`，在 switch 里加：

```csharp
case MotionCommandType.SetAcceleration:
    // 你的硬件SDK.SetAcceleration(command.Speed);
    break;
```

### 第 4 步：前端调用

```csharp
_hub.SetAcceleration(500f);  // 搞定
```

## 七、怎么添加新的工艺模组？

参考现有三个模组的结构，在 ProcessModules 下新建文件夹：

```
ProcessModules\MyNewModule\
├── MyNewModuleProcessModule.cs   ← 继承 ProcessModuleBase
├── MyNewModuleGlobalSetting.cs   ← 全局参数
├── MyNewModuleProjectSetting.cs  ← 项目参数
└── RunForm.cs / .Designer.cs     ← 运行界面
```

然后在 `ProcessModuleManager.cs` 的 `InitAll()` 中注册：

```csharp
ok = RegisterAndInit(new MyNewModuleProcessModule(), "MyNewModule") && ok;
```

## 八、常见问题

**Q：编译报错找不到 `ProcessModuleBase`？**
A：检查是不是把 6 个临时文件删了但还没加 DLL 引用。

**Q：编译报错找不到 `XYView`？**
A：确认 `Controls\` 下的 6 个 .cs 文件都在 csproj 中注册了。

**Q：界面不显示位置？**
A：确认后端服务已注入（`InjectServiceToAll`），且硬件有真实反馈。

**Q：RunForm 能拖动吗？**
A：不能。RunForm 以 `TopLevel=false` 嵌入到 ModuleSettingForm 的 TabPage 中，已失去窗口特性。

**Q：怎么打开设计器编辑界面？**
A：先编译项目（Ctrl+Shift+B），然后右键 RunForm.cs → 查看设计器。
