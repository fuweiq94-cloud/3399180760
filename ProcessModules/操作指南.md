## 一、这个项目是什么？

ProcessModules 是一个**独立的工艺模组类库**，用于控制 **XYZU 四轴**运动，可编译为 `ProcessModules.dll`，挂到上位机平台上作为功能插件使用。

| 角色 | 说明 |
|------|------|
| **上位机平台** | 主程序：调度模组、接硬件、管项目/报警 |
| **工艺模组（本库）** | 插件：主控制 / 点位跳转 / 轨迹查看 + 运行界面 |
| **运动服务** | 实现 `IMotionService`，把标准指令翻译成控制卡/PLC 调用 |

**内置三个模组**

| 注册名 | 功能 |
|--------|------|
| `MainControl` | 手动操控：滑块、JOG、急停、GOTO/ORIGIN/CENTER |
| `PointJump` | 预设点收藏与跳转：GOTOPOINT / SAVEPOINT / DELETEPOINT |
| `TrajectoryView` | 轨迹记录与显示：CLEARTRAIL / SHOWTRAIL / RANDOM |

**依赖关系（当前状态）**

- 业务层 `Logic\`、控件 `Controls\`、各模组界面均在本项目内，**不引用** `XyzController` 主工程。
- 模组基类等来自平台 DLL：`InterfaceDefine`、`MainModule`（代码已 `using` 这两个命名空间）。
- 6 个临时替代源文件**已删除**；若编译报找不到 `ProcessModuleBase` 等类型，需按 [第四节](#四平台-dll-引用配置必做) 添加 DLL 引用。

## 二、怎么打开和编译？

### 方法 1：Visual Studio（推荐）

1. 双击打开 `ProcessModules\ProcessModules.sln`
2. 确认已按 [第四节](#四平台-dll-引用配置必做) 配置 `InterfaceDefine.dll` / `MainModule.dll`
3. 按 `Ctrl + Shift + B` 生成
4. 输出出现 `ProcessModules -> ...\bin\Debug\ProcessModules.dll` 即成功

### 方法 2：命令行 MSBuild

```powershell
cd d:\zm\C#\XyzController\ProcessModules
# 按本机 VS 安装路径调整 MSBuild 位置
& "D:\Program Files\Microsoft Visual Studio\18\Community\MSBuild\Current\Bin\MSBuild.exe" `
  ProcessModules.sln /p:Configuration=Debug
```

### 环境要求

- .NET Framework 4.6.1+
- Visual Studio（含「.NET 桌面开发」工作负载）或独立 MSBuild
- 平台提供的 `InterfaceDefine.dll`、`MainModule.dll`

## 四、平台 DLL 引用配置（必做）

临时替代源文件已移除，编译前必须引用真实平台 DLL。

### 第 1 步：放入 DLL

将平台提供的文件复制到 `ProcessModules\` 目录（或统一 `lib\` 目录，路径与 HintPath 一致即可）：

```
ProcessModules\
├── InterfaceDefine.dll
└── MainModule.dll
```

> 也可从 `DOMOPlatform` 或现场上位机安装目录获取，以现场版本为准。

### 第 2 步：修改 csproj 添加引用

用编辑器打开 `ProcessModules.csproj`，在 BCL 的 `<Reference>` 所在 `ItemGroup` 中增加：

```xml
<Reference Include="InterfaceDefine">
  <HintPath>InterfaceDefine.dll</HintPath>
  <Private>True</Private>
</Reference>
<Reference Include="MainModule">
  <HintPath>MainModule.dll</HintPath>
  <Private>True</Private>
</Reference>
```

若 DLL 放在子目录，例如 `lib\`：

```xml
<HintPath>lib\InterfaceDefine.dll</HintPath>
```

### 第 3 步：重新编译

`Ctrl + Shift + B`。成功后 `bin\Debug\` 下应有 `ProcessModules.dll`，并通常会复制依赖 DLL。

### 第 4 步：代码侧无需改 using

源码中已使用：

```csharp
using InterfaceDefine;
using MainModule;
using ProcessModules;
```

命名空间与 DOMO 模板一致，**不要**再改回本地临时类。

### 迁移状态速查

| 项 | 状态 |
|----|------|
| 删除 6 个临时 .cs | ✅ 已完成 |
| csproj 去掉临时 Compile | ✅ 已完成 |
| 添加 InterfaceDefine / MainModule 引用 | ⬜ 按本节配置 |

## 五、上位机怎么把模组跑起来？（最短路径）

完整六步与可复制 Bootstrapper 见《上位机平台启动指南.md》。最短流程：

```csharp
// 1. 连接硬件（实现 IMotionService，骨架见 PlatformMotionService）
IMotionService svc = new PlatformMotionService();
if (!svc.Open()) return;

// 2. 初始化三个内置模组
ProcessModuleManager.InitAll();

// 3. 注入服务（漏了会出现「界面能动、机器不动」）
ProcessModuleManager.InjectServiceToAll(svc);

// 4. 订阅报警
ProcessModuleManager.SubscribeAlarms(OnModuleAlarm);

// 5. 显示界面（panelHost 为平台上的 Panel）
ProcessModuleBase main = ProcessModuleManager.Get("MainControl");
if (main != null && main.bInitOK)
    main.ShowRunForm(panelHost);

// 6. 可选：切换项目
// ProcessModuleEnvironment.SwitchProject(@"D:\Projects\工件A");
```

**关闭顺序（倒序）**

```csharp
ProcessModuleManager.UnsubscribeAlarms(OnModuleAlarm);
ProcessModuleManager.SaveAll();
ProcessModuleManager.CloseAll();
svc.CloseService();
```

**下发命令示例**

```csharp
// 返回：0=成功，-1=未知命令，-2=参数错误
ProcessModuleManager.Get("PointJump").Action("GOTOPOINT", "A工位");
ProcessModuleManager.Get("MainControl").Action("GOTO", 10f, 20f, 5f);
ProcessModuleManager.Get("MainControl").Action("ESTOP");
```

命令全表、参数与返回值见《接口文档.md》第 11 节。

## 六、怎么对接真实硬件？

### 核心概念

```
硬件驱动代码 → 实现 IMotionService → InjectServiceToAll → 界面显示真实位置
```

### 操作步骤

1. 复制或修改 `Logic\PlatformMotionService.cs`，实现真实 SDK 调用  
2. 在 `ExecuteCommand` 中按 `MotionCommandType` 分发（移动/回原/急停/设速等）  
3. 在硬件位置回调中构造 `AxisPosition`，调用 `RaisePositionUpdated(位置)`  
4. 平台启动时：`Open` → `InitAll` → `InjectServiceToAll`  

```csharp
var svc = new PlatformMotionService(); // 或你的 MyMotionCardService
svc.Open();
ProcessModuleManager.InitAll();
ProcessModuleManager.InjectServiceToAll(svc);
```

### 绝对不能做的事

> ❌ **严禁伪造位置数据**  
> `AxisPosition.Actual` 必须来自编码器/光栅尺/PLC 等真实反馈。  
> 硬件未连接时位置保持 0 或上次真实值，**不要**编造轨迹让界面「看起来在动」。  
> 本地调试可用回环假服务，**仅限开发环境**。

接口字段、指令类型、完整实现模板见《接口文档.md》第 7～9、15 节。

## 七、怎么添加新的运动指令？

例如增加「设置加速度」：

### 第 1 步：枚举加类型

`Logic\MotionCommand.cs`：

```csharp
public enum MotionCommandType
{
    // ... 现有 ...
    SetAcceleration,   // 新增
}
```

### 第 2 步：Hub 加便捷方法

`Logic\XyzControllerHub.cs`：

```csharp
public void SetAcceleration(float acc)
{
    MotionCommand cmd = new MotionCommand(MotionCommandType.SetAcceleration);
    cmd.Speed = acc;   // 或扩展字段 / Tag
    SendCommand(cmd);  // 注意：SendCommand 若为 private，需在 Hub 内调用
}
```

> 若 `SendCommand` 为 private，把新方法写在 `XyzControllerHub` 类内部即可（与现有 `SetTarget` 同级）。

### 第 3 步：后端 switch 处理

`PlatformMotionService.ExecuteCommand`（或你的实现）：

```csharp
case MotionCommandType.SetAcceleration:
    // MotionCardSdk.SetAcceleration(command.Speed);
    break;
```

### 第 4 步：调用

```csharp
_hub.SetAcceleration(500f);
// 或在模组 Action 中增加对应命令名再转发到 Hub
```

## 八、怎么添加新的工艺模组？

### 第 1 步：按现有结构建目录

```
ProcessModules\MyNewModule\
├── MyNewModuleProcessModule.cs   ← 继承 ProcessModuleBase（来自 InterfaceDefine）
├── MyNewModuleGlobalSetting.cs   ← 全局参数
├── MyNewModuleProjectSetting.cs  ← 项目参数
└── RunForm.cs / .Designer.cs / .resx
```

可对照 `PointJump\` 或 `MainControl\` 复制后改名。

### 第 2 步：实现 DOMO 风格生命周期

模组主类至少实现（与 `DOMO.CS` / 现有 `*ProcessModule` 一致）：

- `Init` / `LoadSetting` / `Save` / `ReOpen` / `Close`
- `ShowRunForm` / `ShowSettingForm`
- `Action` / `SetParam` / `GetParam`
- `StopAll` / `ReleaseAlarm` / `GetInfo` / `FunctionCaller`
- `SetMotionService`（转发到内部 Hub）

### 第 3 步：注册到管理器

`ProcessModuleManager.InitAll()` 中增加：

```csharp
ok = RegisterAndInit(new MyNewModuleProcessModule(), "MyNewModule") && ok;
```

### 第 4 步：csproj 加入编译项

```xml
<Compile Include="MyNewModule\MyNewModuleProcessModule.cs" />
<Compile Include="MyNewModule\MyNewModuleGlobalSetting.cs" />
<Compile Include="MyNewModule\MyNewModuleProjectSetting.cs" />
<Compile Include="MyNewModule\RunForm.cs">
  <SubType>Form</SubType>
</Compile>
<Compile Include="MyNewModule\RunForm.Designer.cs">
  <DependentUpon>RunForm.cs</DependentUpon>
</Compile>
```

### 第 5 步：平台使用

```csharp
ProcessModuleManager.Get("MyNewModule").ShowRunForm(panel);
ProcessModuleManager.Get("MyNewModule").Action("你的命令", ...);
```

| 模组 | 常用命令 |
|------|----------|
| **MainControl** | `GOTO x y z` · `ORIGIN` · `CENTER` · `JOG 轴 ±1` · `SETSPEED n` · `STOP`/`ESTOP` |
| **PointJump** | `GOTO` · `GOTOPOINT 名` · `SAVEPOINT 名` · `DELETEPOINT 名` · `SETSPEED` · `STOP` |
| **TrajectoryView** | `GOTO` · `ORIGIN`/`CENTER`/`RANDOM` · `CLEARTRAIL` · `SHOWTRAIL on/off` · `SETSPEED` · `STOP` |

```csharp
main.Action("GOTO", 10f, 20f, 5f);
```

## 十、常见问题 FAQ

**Q：编译报错找不到 `ProcessModuleBase` / `MachineStatus`？**  
A：临时源文件已删，必须引用 `InterfaceDefine.dll` 与 `MainModule.dll`（见第四节）。检查 HintPath 是否指向真实文件。

**Q：编译报错找不到 `XYView` / `IMotionService`？**  
A：确认 `Controls\`、`Logic\` 下对应 `.cs` 已在 csproj 的 `<Compile Include=...>` 中。

**Q：界面能动，机器不动？**  
A：99% 漏了 `InjectServiceToAll`，或 `IMotionService.Open()` 失败（`IsConnected == false`）。

**Q：界面位置不刷新？**  
A：后端未触发 `PositionUpdated`，或 `Actual` 未写入真实硬件值。

**Q：收不到报警？**  
A：是否调用了 `SubscribeAlarms`；handler 签名应为  
`void Handler(object sender, ModuleAlarmEventArgs e)`。

**Q：`Get("MainControl")` 返回 null？**  
A：是否先调用了 `InitAll()`；注册名是否拼写正确（不区分大小写）。

**Q：RunForm 能拖动吗？**  
A：不能。以 `TopLevel=false` 嵌入 Panel，`Dock=Fill`，已失去独立窗口特性。

**Q：怎么打开 WinForms 设计器？**  
A：先成功编译一次，再右键 `RunForm.cs` → 查看设计器。设计器打不开时优先检查 Designer 与引用是否编译通过。

**Q：没有硬件怎么测界面？**  
A：可写回环服务：在 `ExecuteCommand` 后把目标坐标通过 `RaisePositionUpdated` 推回。仅限本地调试，生产禁用。
