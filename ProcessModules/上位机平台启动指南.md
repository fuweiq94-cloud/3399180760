## 第一部分：先搞懂几个概念（术语扫盲）

别急着写代码，先花 5 分钟理解下面几个词，后面看代码会轻松很多。

### 1.1 什么是「上位机平台」？

**上位机**就是「上面那台机器/软件」——在工业现场，它通常指**主控电脑上的主程序**，负责：

- 显示总界面、调度各个功能模块  
- 跟硬件（运动控制卡、PLC）打交道  
- 管理项目、报警、权限等  

而**工艺模组**（本类库）是「插在上位机上的功能插件」。上位机是「主机」，工艺模组是「插件」。

### 1.2 什么是「工艺模组」？

一个**工艺模组 = 一个独立的功能页面 + 它背后的控制逻辑**。本项目内置了 3 个：

| 注册名 | 大白话 | 界面上能干什么 |
|--------|--------|----------------|
| **MainControl** | 手动操控盘 | 拖滑块、按 JOG 按钮、急停，手动移动 X/Y/Z 轴 |
| **PointJump** | 收藏夹跳转 | 把常用位置存成「预设点」，点一下自动跳过去 |
| **TrajectoryView** | 运动轨迹记录仪 | 显示并记录 XYZ 走过的路径 |

每个模组都有**两张脸**：

- **运行界面 RunForm**：平时操作的主界面（`TopLevel=false` 嵌入平台 Panel）  
- **设置界面**：改参数用（如 `ModuleSettingForm` / 各模组 SettingForm）  

> 注册名**不区分大小写**（`Get("maincontrol")` 与 `Get("MainControl")` 等价）。

### 1.3 什么是「接口 / 依赖倒置」？（插座类比）

这是整个架构的**核心思想**，务必理解。

> 你家的**插座**（接口）是标准的。不管你插的是电视、冰箱还是手机充电器（具体实现），只要插头符合插座标准，就能通电。  
> 墙里的电线（平台）**不需要知道**你插的是什么电器——它只负责按标准供电。

本项目里：

| 角色 | 对应 |
|------|------|
| **插座（接口）** | `IMotionService`（运动控制标准约定） |
| **电器（实现）** | 具体硬件服务（某品牌控制卡、某型号 PLC） |
| **墙里电线（平台/模组）** | 只认「插座」，不认「具体电器」 |

**好处**：将来换硬件，只要新硬件也实现 `IMotionService`，平台和模组**一行代码都不用改**。这就叫「依赖倒置」。

### 1.4 几个核心类的角色（餐厅类比）

| 类名 | 餐厅角色 | 它负责什么 | 类型来源 |
|------|----------|------------|----------|
| `ProcessModuleManager` | **前台总管** | 平台主要跟它打交道：登记、初始化、注入、报警 | `ProcessModules` |
| `ProcessModuleBase` | **员工手册** | 规定每个模组必须会做什么 | `InterfaceDefine` |
| `XyzControllerHub` | **模组的大脑** | 算坐标、打包指令、收位置反馈 | `ProcessModules` |
| `IMotionService` | **翻译官** | 标准指令 ↔ 硬件 SDK | `ProcessModules`（接口） |
| `RunForm` / 设置窗体 | **模组的脸** | 显示给用户看的界面 | 各模组目录 |
| `ProcessModuleEnvironment` | **广播站** | 广播「换项目了」等消息 | `InterfaceDefine` |
| `ModuleAlarmEventArgs` | **报警单** | 哪个模组、什么内容、什么时间 | `MainModule` |

> 记住一句话：**平台主要跟「前台总管」`ProcessModuleManager` 说话就行**，不用直接指挥每个模组。

## 第二部分：整体流程，一图看懂

### 2.1 启动流程（平台要做的事）

```
平台启动
   │
   ├─ 第1步：连接硬件后端（创建 IMotionService 并 Open）
   │
   ├─ 第2步：初始化所有模组（ProcessModuleManager.InitAll）
   │
   ├─ 第3步：把硬件服务「插」进所有模组（InjectServiceToAll）★关键
   │
   ├─ 第4步：接好报警线（SubscribeAlarms）
   │
   ├─ 第5步：把模组界面显示到面板上（ShowRunForm）
   │
   └─ 第6步：打开项目（SwitchProject，可选）
```

### 2.2 关闭流程（倒序）

```
UnsubscribeAlarms → SaveAll → CloseAll → CloseService
```

### 2.3 数据怎么流动（运行起来之后）

```
用户在界面拖滑块 / 平台调用 Action
      │
      ▼
模组大脑 XyzControllerHub.SetTarget(目标坐标)
      │ 把目标打包成「标准指令」MotionCommand
      ▼
翻译官 IMotionService.ExecuteCommand(指令)
      │ 翻译成硬件听得懂的话
      ▼
真实硬件（控制卡/PLC）开始运动
      │ 硬件实时反馈「我现在到哪了」
      ▼
翻译官 PositionUpdated 事件（推送 AxisPosition）
      │
      ▼
模组大脑更新 Current → Hub.Changed → 界面刷新 DRO / 视图
```

> 界面**从不直接碰硬件**，一切都经过「大脑」和「翻译官」。换硬件时界面不用改。  
> ⚠️ **铁律**：`AxisPosition.Actual` 必须来自真实硬件，严禁生产环境伪造位置。

## 第三部分：手把手六步启动

下面每一步都按「干什么 → 为什么 → 代码」来讲。

### 第 1 步：连接硬件后端

**干什么**：创建一个「翻译官」（`IMotionService` 的实现），并打开它和硬件的连接。

**为什么**：模组发出的指令需要有人翻译成硬件语言；硬件的位置反馈也需要有人传回来。

**怎么做**：

```csharp
// 类库已提供「翻译官骨架」PlatformMotionService，
// 真实项目里复制它、填入你硬件的 SDK 调用（见《接口文档.md》第 7、15 节）。
IMotionService motionService = new PlatformMotionService();

// 打开连接（内部会初始化控制卡 / 建立 PLC 通信）
if (!motionService.Open())
{
    // Open 返回 false 说明连不上硬件
    MessageBox.Show("运动控制后端连接失败，请检查硬件");
    return; // 连不上就别往下走了
}
```

> 💡 **没有真实硬件怎么办？** 见 FAQ Q4。

---

### 第 2 步：初始化所有模组

**干什么**：让「前台总管」把内置的 3 个模组都登记并初始化好。

**为什么**：模组初始化时会读取参数文件、创建内部 Hub、注册项目切换事件。必须先初始化，后面才能用。

**怎么做**：

```csharp
// 一次性创建并初始化 MainControl / PointJump / TrajectoryView
bool ok = ProcessModuleManager.InitAll();

if (!ok)
{
    // 有模组初始化失败（常见：参数文件损坏、DLL 类型不匹配）
    MessageBox.Show("部分工艺模组初始化失败，请检查参数文件与平台 DLL");
}
```

> 💡 `InitAll()` 是**幂等**的——调多次和调一次效果一样。  
> 💡 初始化时 Hub 的服务引用为 `null`，此时界面可显示，但**指令还到不了硬件**（见第 3 步）。

---

### 第 3 步：把硬件服务「插」进所有模组（★最关键）

**干什么**：把第 1 步的「翻译官」交给每一个模组。

**为什么**：模组初始化时，大脑（Hub）里「翻译官」位置是**空的**（`null`）。只有把翻译官插进去，指令才能真正发到硬件。  
**这一步漏了，就会出现「界面有反应、机器不动」。**

**怎么做**：

```csharp
// 一行代码注入全部模组（内部逐个调用 SetMotionService → Hub.SetService）
ProcessModuleManager.InjectServiceToAll(motionService);
```

> 💡 注入后 Hub 会：退订旧服务 → 订阅新服务的 `PositionUpdated` → 立即 `GetPosition()` 同步一次位置。  
> 💡 运行期换硬件：再调一次 `InjectServiceToAll(新服务)` 即可。

---

### 第 4 步：接好报警线

**干什么**：订阅所有模组的「报警事件」，模组出错时平台能收到通知。

**为什么**：命令写错、参数不对、预设点不存在等会 `InsertAlarm`。不订阅则报警「石沉大海」。

**怎么做**：

```csharp
ProcessModuleManager.SubscribeAlarms(OnModuleAlarm);

private void OnModuleAlarm(object sender, ModuleAlarmEventArgs e)
{
    // e.ModuleName = 哪个模组（如 "PointJump"）
    // e.Message    = 报警内容（如 "未知命令:XXX"）
    // e.Time       = 报警时间
    // TODO: 转发到平台报警列表 / 弹窗 / 日志
    Console.WriteLine("【报警】{0}：{1}（{2}）", e.ModuleName, e.Message, e.Time);
}
```

> 字段以 `MainModule.dll` 中 `ModuleAlarmEventArgs` 实际定义为准。

---

### 第 5 步：把模组界面显示到面板上

**干什么**：在平台界面上放一个 `Panel`，让模组把自己的 RunForm「填」进去。

**为什么**：RunForm 需要容器；平台提供 Panel，模组自动 `Dock=Fill` 铺满。

**怎么做**：

```csharp
ProcessModuleBase main = ProcessModuleManager.Get("MainControl");

if (main != null && main.bInitOK)
{
    // panelHost 是你平台界面上的 Panel，需有有效宽高
    main.ShowRunForm(panelHost);
}
```

> 💡 `ShowRunForm` 内部：`TopLevel=false`、清空 panel、`Dock=Fill`、`Show()`。  
> 💡 设置界面：`main.ShowSettingForm(另一个Panel)`。  
> 💡 切换模组：对同一 Panel 再 `Get("PointJump").ShowRunForm(panelHost)` 即可。

---

### 第 6 步：打开项目（可选）

**干什么**：告诉所有模组「现在用哪个项目」，模组自动加载该项目参数。

**为什么**：不同产品/工件有不同参数（如预设点位）。切换项目时要重载。

**怎么做**：

```csharp
// 切换到指定项目路径
ProcessModuleEnvironment.SwitchProject(@"D:\Projects\工件A");

// 或使用当前默认路径
ProcessModuleEnvironment.SwitchProject(ProcessModuleEnvironment.CurrentProjectPath);
```

> 模组在 `Init` 时已订阅 `OpenProject` 事件，收到后会 `LoadSetting()`。

## 第四部分：完整可复制的示例代码

把上面六步串起来，就是一个「平台启动器」。可直接复制到平台工程：

```csharp
using System;
using System.Windows.Forms;
using InterfaceDefine;     // ProcessModuleBase、ProcessModuleEnvironment
using MainModule;          // ModuleAlarmEventArgs
using ProcessModules;      // ProcessModuleManager、IMotionService、PlatformMotionService

/// <summary>
/// 上位机平台「工艺模组启动器」。
/// 平台启动时调 Start()，关闭时调 Shutdown()。
/// </summary>
public class PlatformBootstrapper
{
    private IMotionService _motionService;

    /// <param name="runPanel">显示运行界面的 Panel</param>
    /// <param name="settingPanel">显示设置界面的 Panel（可 null）</param>
    public bool Start(Panel runPanel, Panel settingPanel)
    {
        // —— 第1步：连接硬件后端 ——
        _motionService = new PlatformMotionService(); // TODO: 换成真实硬件实现
        if (!_motionService.Open())
            return false;

        // —— 第2步：初始化所有模组 ——
        if (!ProcessModuleManager.InitAll())
            return false;

        // —— 第3步：注入硬件服务 ★ ——
        ProcessModuleManager.InjectServiceToAll(_motionService);

        // —— 第4步：接报警 ——
        ProcessModuleManager.SubscribeAlarms(OnModuleAlarm);

        // —— 第5步：显示主控制运行界面 ——
        ProcessModuleBase main = ProcessModuleManager.Get("MainControl");
        if (main != null && main.bInitOK && runPanel != null)
            main.ShowRunForm(runPanel);

        if (main != null && main.bInitOK && settingPanel != null)
            main.ShowSettingForm(settingPanel);

        // —— 第6步：打开默认项目（可选）——
        ProcessModuleEnvironment.SwitchProject(
            ProcessModuleEnvironment.CurrentProjectPath);

        return true;
    }

    /// <summary>平台关闭时调用。</summary>
    public void Shutdown()
    {
        // 顺序：退订报警 → 保存 → 关模组 → 关硬件
        ProcessModuleManager.UnsubscribeAlarms(OnModuleAlarm);
        ProcessModuleManager.SaveAll();
        ProcessModuleManager.CloseAll();
        if (_motionService != null)
            _motionService.CloseService();
    }

    private void OnModuleAlarm(object sender, ModuleAlarmEventArgs e)
    {
        // TODO: 转发到平台报警系统
        Console.WriteLine("【报警】{0}：{1}（{2}）", e.ModuleName, e.Message, e.Time);
    }
}
```

**在平台主窗体里这样用**：

```csharp
public partial class PlatformMainForm : Form
{
    private PlatformBootstrapper _bootstrapper = new PlatformBootstrapper();

    private void PlatformMainForm_Load(object sender, EventArgs e)
    {
        // panelRun 是界面上预先放好的 Panel
        bool ok = _bootstrapper.Start(panelRun, null);
        if (!ok)
            MessageBox.Show("工艺模组启动失败");
    }

    private void PlatformMainForm_FormClosing(object sender, FormClosingEventArgs e)
    {
        _bootstrapper.Shutdown();
    }
}
```

## 第五部分：平台运行起来后能做什么

### 5.1 下发命令（让模组干活）

用模组的 `Action` 方法：第一个参数是「命令名」，后面是参数。

```csharp
ProcessModuleBase pointJump = ProcessModuleManager.Get("PointJump");

// 跳到坐标 (10, 20, 5)
int ret = pointJump.Action("GOTO", 10f, 20f, 5f);

// 返回值：0=成功，-1=未知/空命令，-2=参数错误或业务失败
if (ret != 0)
    Console.WriteLine("命令失败，错误码：" + ret);
```

### 5.2 三个模组命令速查表

> 命令名**不区分大小写**（`"goto"` / `"GOTO"` 都行）。

**MainControl（主控制）**

| 命令 | 例子 | 作用 |
|------|------|------|
| `GOTO x y z` | `Action("GOTO", 10f, 20f, 5f)` | 三轴移动到指定坐标 |
| `ORIGIN` | `Action("ORIGIN")` | 回原点 (0,0,0) |
| `CENTER` | `Action("CENTER")` | 回到行程中心 |
| `JOG 轴 方向` | `Action("JOG", "X", 1)` | X 轴正向寸动一步（方向 +1/-1） |
| `SETSPEED 值` | `Action("SETSPEED", 80)` | 设速度档位（0~100） |
| `STOP` / `ESTOP` | `Action("ESTOP")` | 停止 / 急停 |

**PointJump（点位跳转）**

| 命令 | 例子 | 作用 |
|------|------|------|
| `GOTO x y z` | `Action("GOTO", 10f, 20f, 5f)` | 跳到指定坐标 |
| `GOTOPOINT 名字` | `Action("GOTOPOINT", "A工位")` | 跳到预设点 |
| `SAVEPOINT 名字` | `Action("SAVEPOINT", "P1")` | 把当前目标存为预设点 |
| `DELETEPOINT 名字` | `Action("DELETEPOINT", "P1")` | 删除预设点 |
| `SETSPEED 值` | `Action("SETSPEED", 60)` | 设速度档位 |
| `STOP` | `Action("STOP")` | 冻结所有轴目标到当前值 |

**TrajectoryView（轨迹查看）**

| 命令 | 例子 | 作用 |
|------|------|------|
| `GOTO x y z` | `Action("GOTO", 10f, 20f, 5f)` | 移动并记录轨迹 |
| `ORIGIN` / `CENTER` / `RANDOM` | `Action("RANDOM")` | 回原点 / 回中心 / 随机目标 |
| `CLEARTRAIL` | `Action("CLEARTRAIL")` | 清除轨迹 |
| `SHOWTRAIL on/off` | `Action("SHOWTRAIL", "on")` | 开/关轨迹显示（`true`/`1` 也可开） |
| `SETSPEED 值` | `Action("SETSPEED", 50)` | 设速度档位 |
| `STOP` | `Action("STOP")` | 冻结所有轴 |

完整参数校验与报警消息见《接口文档.md》第 11 节。

### 5.3 读写参数

```csharp
ProcessModuleBase main = ProcessModuleManager.Get("MainControl");

main.SetParam("SpeedSetting", 80);              // 速度档位
main.SetParam("JogStep", 1.0f);                 // 寸动步长
main.SetParam("JogIncremental", true);          // true=寸动，false=连续

float x = Convert.ToSingle(main.GetParam("CurrentX"));  // 实际位置
float tx = Convert.ToSingle(main.GetParam("TargetX"));  // 目标位置
int spd = Convert.ToInt32(main.GetParam("SpeedSetting"));
```

**常用 GetParam 键**

| 模组 | 键 | 含义 |
|------|-----|------|
| 通用 | `CurrentX/Y/Z`、`SpeedSetting` | 实际位置、速度 |
| MainControl | `TargetX/Y/Z` | 目标位置 |
| PointJump | `PresetCount`、`JumpCount` | 预设数量、跳转次数 |
| TrajectoryView | `ShowTrail`、`TrailPointCount` | 是否显示轨迹、轨迹点数 |

### 5.4 切换不同模组的界面

```csharp
// 同一 Panel 切换
ProcessModuleManager.Get("PointJump").ShowRunForm(panelRun);
ProcessModuleManager.Get("TrajectoryView").ShowRunForm(panelRun);

// 多 Panel 同时显示（一个 Panel 同时只放一个模组）
ProcessModuleManager.Get("MainControl").ShowRunForm(panel1);
ProcessModuleManager.Get("PointJump").ShowRunForm(panel2);
```

### 5.5 统一停止 / 保存

```csharp
ProcessModuleManager.StopAll();   // 所有模组停止运动/流程
ProcessModuleManager.SaveAll();   // 保存所有模组参数
```

### 5.6 运行期更换硬件服务

```csharp
IMotionService newSvc = new MyOtherCardService();
if (newSvc.Open())
    ProcessModuleManager.InjectServiceToAll(newSvc);
```

## 第六部分：如何正确关闭

关闭顺序很重要，**倒着来**（先关上层，再关硬件）：

```csharp
ProcessModuleManager.UnsubscribeAlarms(OnModuleAlarm);  // 1. 先退订报警
ProcessModuleManager.SaveAll();                          // 2. 保存参数
ProcessModuleManager.CloseAll();                         // 3. 关闭所有模组
motionService.CloseService();                            // 4. 最后关硬件连接
```

> 顺序错了可能导致：报警重复触发、参数没保存、硬件资源没释放。

## 第七部分：常见问题 FAQ

**Q1：界面没显示出来？**  
- `Get("MainControl")` 是否为 `null`（名字拼错？没先 `InitAll`？）  
- `bInitOK` 是否为 `true`  
- `Panel` 是否在界面上、宽高是否为 0  

**Q2：界面能动，但机器不动？**  
- 99% 漏了第 3 步 `InjectServiceToAll`  
- 或 `Open()` 失败（`IsConnected == false`）  
- 或 `ExecuteCommand` 里 SDK 未真正调用  

**Q3：收不到报警？**  
- 是否调用了 `SubscribeAlarms`  
- 签名是否为：`void 方法名(object sender, ModuleAlarmEventArgs e)`  

**Q4：没有真实硬件，怎么测试界面？**  
- 复制 `PlatformMotionService`，在 `ExecuteCommand` 后把目标坐标通过 `RaisePositionUpdated` 回推  
- ⚠️ 回环假服务**仅限本地调试**，生产严禁模拟位置  

**Q5：编译报错「找不到 ProcessModuleManager / IMotionService」？**  
- 检查三行 `using`（见文首）  
- 平台工程是否引用了 `ProcessModules.dll`  

**Q6：编译报错「找不到 ProcessModuleBase / ModuleAlarmEventArgs」？**  
- 平台工程是否同时引用了 `InterfaceDefine.dll`、`MainModule.dll`  
- 若是编译 ProcessModules 本身，见《操作指南.md》第四节  

**Q7：怎么知道有哪些模组、叫什么名字？**  
- 固定注册名：`"MainControl"`、`"PointJump"`、`"TrajectoryView"`  
- 或遍历 `ProcessModuleManager.Modules`  

**Q8：能让两个模组同时显示吗？**  
- 可以。每个模组各一个 `Panel`，分别 `ShowRunForm`  

**Q9：Action 返回 true 了轴却没到位？**  
- `Action` 返回 `0` 只表示命令被模组接受并下发；是否到位看位置反馈（`Current` / `IsMoving`）  
- `ExecuteCommand` 返回 `true` 同理，只表示硬件接受指令  

**Q10：位置不刷新？**  
- 后端是否触发 `PositionUpdated`  
- `Actual` 是否写入真实硬件值

### 8.2 平台集成 API 速查

| 方法 | 作用 |

| `Modules` | 遍历所有模组 |
| `InjectServiceToAll(服务)` | 给所有模组注入硬件服务 ★ |
| `SubscribeAlarms(处理方法)` | 统一订阅报警 |
| `UnsubscribeAlarms(处理方法)` | 统一取消报警 |
| `StopAll()` / `SaveAll()` / `CloseAll()` | 统一停止 / 保存 / 关闭 |

| 方法/属性 | 作用 |
|-----------|------|
| `SwitchProject(路径)` | 切换项目并广播 |
| `CurrentProjectPath` | 当前项目路径 |
| `AppParamPath()` | 全局参数根目录 |

**ProcessModuleBase（每个模组）**

| 方法/属性 | 作用 |
|-----------|------|

| `Action(命令, 参数...)` | 下发命令（0=成功，-1=未知，-2=参数错） |
| `SetParam(键, 值)` / `GetParam(键)` | 写 / 读参数 |
| `bInitOK` | 是否初始化成功 |
| `StopAll()` / `Save()` / `Close()` | 停止 / 保存 / 关闭 |

**IMotionService（翻译官）**
