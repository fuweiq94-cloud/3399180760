## 第一部分：先搞懂三个核心概念

工艺模组要和硬件打交道，靠的是下面三个「约定」。先用类比理解它们。

### 1.1 IMotionService —— 「翻译官」（插座标准）

模组说的是「标准普通话」（统一指令），但不同硬件听的是「方言」（各家 SDK 不一样）。
**`IMotionService` 就是中间的翻译官**：

```
模组（说普通话）  →  翻译官 IMotionService  →  硬件（听方言）
                          ↑
              不同品牌硬件，配不同翻译官
```

它是一个**接口（interface）**，也就是「插座标准」。无论你接的是 A 品牌控制卡还是 B 品牌 PLC，只要你的翻译官实现了这个接口，模组就能用——**模组完全不关心你接的是什么硬件**。

### 1.2 MotionCommand —— 「标准指令单」

模组不会直接喊「X 轴走到 50」，而是填一张**标准指令单**（`MotionCommand`），上面写明：
- 指令类型（绝对移动？回原点？急停？）
- 目标位置
- 速度
- ……

翻译官拿到指令单，按类型翻译成硬件听得懂的话。

> 为什么用「统一指令单」？因为这样**只有一个入口**（`ExecuteCommand`），将来换硬件、加指令都只改一处，好维护。

### 1.3 AxisPosition —— 「位置快照」

硬件运动时，要不断告诉前端「我现在到哪了」。`AxisPosition` 就是某一时刻的**位置快照**，包含：
- 各轴的**目标位置**（想去哪）
- 各轴的**实际位置**（真实在哪，来自编码器/光栅尺）
- 各轴是否在运动、是否已回原、是否报警

> ⚠️ **铁律**：实际位置（`Actual`）**必须来自真实硬件反馈**，严禁用模拟/伪造数据。这是为了保证界面显示和真实机器一致，避免安全事故。

### 1.4 三者关系一图流

```
模组大脑 Hub
   │ 填一张 MotionCommand（标准指令单）
   ▼
IMotionService.ExecuteCommand(指令单)   ← 翻译官
   │ 翻译成硬件方言
   ▼
真实硬件运动
   │ 硬件反馈位置
   ▼
IMotionService 触发 PositionUpdated 事件，带上 AxisPosition（位置快照）
   │
   ▼
模组大脑 Hub 更新各轴 Current → 界面刷新
```

## 第二部分：整体架构（数据怎么流）

系统分三层，**前端界面绝不直接碰硬件**：

```
┌─────────────────────────────────────────────────────────┐
│  ① 前端界面层（RunForm / ModuleSettingForm）              │
│     - 只读取 Hub 里的 Target / Current 来显示             │
│     - 只通过 Hub 下发目标，不直接接触硬件                  │
└────────────────────────┬────────────────────────────────┘
                         │ 订阅 Hub.Changed（位置变了就刷新）
┌────────────────────────▼────────────────────────────────┐
│  ② 业务大脑层（XyzControllerHub）                         │
│     - 持有翻译官 IMotionService 的引用                    │
│     - 设目标 → 打包成 MotionCommand → 交给翻译官          │
│     - 收到位置反馈 → 写入各轴 Current → 通知前端          │
└────────────────────────┬────────────────────────────────┘
                         │ IMotionService（插座标准）
┌────────────────────────▼────────────────────────────────┐
│  ③ 后端服务层（IMotionService 的具体实现）                │
│     - ExecuteCommand：收到标准指令 → 转发给硬件           │
│     - PositionUpdated：硬件反馈 → 推送真实位置            │
│     - 严禁模拟位置数据                                    │
└─────────────────────────────────────────────────────────┘
```

**两条数据流**：
- **指令下发**（自上而下）：界面设目标 → Hub 打包指令 → 翻译官 → 硬件
- **位置反馈**（自下而上）：硬件 → 翻译官推位置 → Hub 更新 → 界面刷新

## 第三部分：核心接口 IMotionService 详解

**文件位置**：`ProcessModules\Logic\IMotionService.cs`

### 3.1 接口长这样

```csharp
public interface IMotionService
{
    // ===== 指令下发（前端 → 后端，唯一入口）=====
    bool ExecuteCommand(MotionCommand command);

    // ===== 位置反馈（后端 → 前端，实时推送）=====
    event EventHandler<AxisPositionEventArgs> PositionUpdated;
    AxisPosition GetPosition();

    // ===== 状态 =====
    bool IsConnected { get; }
    int AxisCount { get; }

    // ===== 生命周期 =====
    bool Open();
    void CloseService();
}
```

### 3.2 每个成员大白话解释

| 成员 | 方向 | 大白话 | 你要做什么 |
|------|------|--------|------------|
| `ExecuteCommand(指令)` | 前端→后端 | **唯一指令入口**。移动/回原/急停/设速度…全走这里 | 按指令类型，调对应硬件 SDK |
| `PositionUpdated` 事件 | 后端→前端 | 硬件位置变了，用它**主动通知**前端 | 在硬件回调里触发这个事件 |
| `GetPosition()` | 前端→后端 | 前端**主动问**「现在在哪」（初始化/重连时用） | 从硬件读一次当前位置返回 |
| `IsConnected` | — | 硬件连上了没？ | 返回通信是否正常 |
| `AxisCount` | — | 管几个轴？（当前是 4：X/Y/Z/U） | 返回轴数量 |
| `Open()` | — | 开机：建立硬件连接 | 初始化控制卡/连 PLC，成功返回 true |
| `CloseService()` | — | 关机：释放硬件资源 | 断开连接、释放资源 |

### 3.3 为什么这样设计？
- **指令只有一个入口**（`ExecuteCommand`）：将来加新指令、换硬件，只改这一处。
- **位置靠「事件推送」**（`PositionUpdated`）：硬件什么时候动、动到哪，由硬件说了算，前端被动接收，保证实时、准确。
- **接口与实现分离**：模组只认接口，换硬件零改动。

## 第四部分：统一指令 MotionCommand 详解

**文件位置**：`ProcessModules\Logic\MotionCommand.cs`

### 4.1 指令类型（每个都解释）

```csharp
public enum MotionCommandType
{
    MoveAbsolute,        // 绝对移动：走到指定坐标
    MoveRelative,        // 相对移动：从当前位置再走一段
    Home,                // 回原点
    Stop,                // 停止（减速停）
    EmergencyStop,       // 急停（立即停）
    SetSpeed,            // 设置速度
    SetCoordinateSystem, // 设置坐标系（机械坐标/工件坐标）
    Enable,              // 使能（给轴上电）
    Disable,             // 去使能（断电）
    Reset                // 复位报警
}
```

### 4.2 指令单的结构（字段解释）

```csharp
public class MotionCommand
{
    public MotionCommandType CommandType;  // 指令类型（上面枚举之一）
    public float[] Positions;              // 目标位置数组 [X, Y, Z, U]（移动类指令用）
    public float Speed;                    // 速度（含义由后端定：mm/s 或档位）
    public int CoordinateSystem;           // 坐标系编号（设坐标系指令用）
    public object Tag;                     // 扩展数据（后端自定义用，如轴掩码、加速度）
}
```

> 💡 不是每个字段每条指令都用。比如「急停」只需要 `CommandType=EmergencyStop`，其他字段可不管。

### 4.3 快捷构造方法（推荐用这些，别手动 new）

```csharp
// 绝对移动到 (X=10, Y=20, Z=5, U=0)，速度 50
MotionCommand cmd1 = MotionCommand.CreateMoveAbsolute(
    new float[] { 10f, 20f, 5f, 0f }, speed: 50f);

// 相对移动：X 再走 +1
MotionCommand cmd2 = MotionCommand.CreateMoveRelative(
    new float[] { 1f, 0f, 0f, 0f });

// 设置速度为 80
MotionCommand cmd3 = MotionCommand.CreateSetSpeed(80f);

// 切换到 1 号工件坐标系
MotionCommand cmd4 = MotionCommand.CreateSetCoordinateSystem(1);
```

## 第五部分：位置数据 AxisPosition 详解

**文件位置**：`ProcessModules\Logic\AxisPosition.cs`

### 5.1 结构字段解释

```csharp
public class AxisPosition
{
    public float[] Target;      // 各轴目标位置 [X, Y, Z, U]（想去哪）
    public float[] Actual;      // 各轴实际位置（真实在哪，★必须来自硬件）
    public bool[] IsMoving;     // 各轴是否正在运动
    public bool[] IsHomed;      // 各轴是否已完成回原点
    public bool[] IsAlarmed;    // 各轴是否处于报警状态
    public DateTime Timestamp;  // 这份数据的时间戳

    public int AxisCount { get; }              // 轴数量（自动按 Actual 长度算）
    public static AxisPosition CreateEmpty(int n);  // 创建 n 轴的全零快照
    public AxisPosition Clone();               // 深拷贝一份
}
```

### 5.2 位置事件参数

位置推送时，用 `AxisPositionEventArgs` 包装一下：

```csharp
public class AxisPositionEventArgs : EventArgs
{
    public AxisPosition Position { get; }   // 最新位置快照
}
```

### 5.3 铁律（务必遵守）

> **严禁在 `GetPosition()` 或 `PositionUpdated` 里返回模拟/伪造的位置数据。**
> `Actual` 数组必须来自真实硬件反馈（编码器、光栅尺、PLC 寄存器）。
> 原因：界面显示的位置必须和真实机器一致，否则可能误判导致撞机、伤人等安全事故。

### 6.2 实现要点（划重点）

1. **`ExecuteCommand` 里按 `CommandType` 分发**——这是指令的总入口。
2. **位置反馈靠 `PositionUpdated` 事件**——在硬件回调（`OnHardwarePosition`）里触发它，前端才会刷新。
3. **`Actual` 必须是真实硬件值**——不能造假（见第五部分铁律）。
4. **`Open`/`CloseService` 管理连接生命周期**——开机初始化、关机释放。

## 第七部分：前端如何获取位置（界面侧）

前端界面**从不直接调硬件**，只通过模组大脑 `XyzControllerHub` 读写：

```csharp
// 在 RunForm 里（这些代码项目里已有，你不用改）：

// 读「实际位置」（来自硬件反馈）
float actualX = _hub.X.Current;
float actualY = _hub.Y.Current;
float actualZ = _hub.Z.Current;
float actualU = _hub.U.Current;

// 读「目标位置」
float targetX = _hub.X.Target;

// 设新目标（Hub 会自动打包成 MotionCommand 发给硬件）
_hub.SetTarget(10f, 20f, 5f, 0f);

// 回原点
_hub.ResetToOrigin();

// 急停
_hub.EmergencyStop();
```

### 位置更新的完整链路

```
硬件反馈
   → IMotionService.PositionUpdated 事件
   → Hub 的回调 Service_PositionUpdated
   → 各轴 AxisController.UpdateCurrent(真实值)
   → Hub.Changed 事件
   → RunForm 刷新 DRO / XYView / ZBarView
```

> 💡 你只要在翻译官里正确触发 `PositionUpdated`，界面就会自动刷新——中间的链路项目已经搭好了。

```csharp
case MotionCommandType.SetAcceleration:

```

完成后，前端调 `_hub.SetAcceleration(500f)` 即可，**界面代码不用改**。

## 第九部分：常见问题 FAQ

**Q1：界面位置不刷新 / 一直不动？**
- 多半是翻译官里**没触发 `PositionUpdated` 事件**。硬件位置变了，要在回调里 `h(this, new AxisPositionEventArgs(位置))`。
- 或者 `Open()` 没成功（`IsConnected` 为 false）。

**Q2：为什么不能用模拟位置？**
- 安全原因：界面显示必须和真实机器一致。如果界面显示「在原点」但机器其实在别处，可能误操作导致撞机/伤人。
- 本地调试界面时可用「回环假服务」临时推位置，但**正式对接必须用真实硬件反馈**。

**Q3：`ExecuteCommand` 返回 false 怎么回事？**
- 返回 false 表示「指令被拒绝」。常见原因：没连接（`IsConnected=false`）、指令为 null、参数非法、硬件不接受。
- 检查你的实现里在什么情况下 return false。

**Q4：位置反馈在哪个线程？界面刷新会卡吗？**
- 硬件回调通常在**通信线程**，不是 UI 线程。Hub 收到位置后触发 `Changed`，界面刷新时若涉及控件操作，需用 `Control.Invoke` 回到 UI 线程（项目里的 RunForm 已处理）。

**Q5：怎么知道指令下发成功没？**
- `ExecuteCommand` 返回 `true` = 硬件接受了指令；`false` = 拒绝。
- 注意：「接受指令」≠「运动完成」。运动是否到位，要看位置反馈（`IsMoving`/`Actual`）。

**Q6：可以中途换硬件吗？**
- 可以。重新 `InjectServiceToAll(新翻译官)` 即可，前端和模组都不用改（Hub 的 `SetService` 会自动退订旧服务、订阅新服务）。

### 10.2 指令类型速查

| 指令类型 | 含义 | 用到的字段 |
|----------|------|------------|
| `MoveAbsolute` | 绝对移动 | `Positions` + `Speed` |
| `MoveRelative` | 相对移动 | `Positions` + `Speed` |
| `Home` | 回原点 | — |
| `Stop` | 停止 | — |
| `EmergencyStop` | 急停 | — |
| `SetSpeed` | 设速度 | `Speed` |
| `SetCoordinateSystem` | 设坐标系 | `CoordinateSystem` |
| `Enable` / `Disable` | 使能 / 去使能 | — |
| `Reset` | 复位报警 | — |

### 10.3 术语表

| 术语 | 解释 |
|------|------|
| IMotionService | 后端运动服务接口（翻译官/插座标准） |
| MotionCommand | 统一运动指令单 |
| AxisPosition | 多轴位置快照 |
| 翻译官 | 把标准指令翻译成具体硬件调用的实现 |
| 位置反馈 | 硬件把真实位置回报给前端 |
| 使能 (Enable) | 给轴上电，让它能受力/运动 |
| 回原 (Home) | 让轴回到机械原点 |
| 坐标系 | 机械坐标(0)/工件坐标(1+)，决定坐标的参考基准 |
| DRO | 数字位置读数显示 |
