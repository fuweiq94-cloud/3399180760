## 2. 上位机平台核心能力

上位机平台通过 `ProcessModules.dll` 获得以下能力：

| 能力 | 说明 | 入口 |
|------|------|------|
| **模组生命周期管理** | 统一初始化 / 停止 / 保存 / 关闭三个内置工艺模组 | `ProcessModuleManager` |
| **硬件服务注入** | 将运动控制后端（控制卡/PLC）注入全部模组 | `InjectServiceToAll` |
| **界面嵌入** | 将模组 RunForm / SettingForm 嵌入平台 Panel | `ShowRunForm` / `ShowSettingForm` |
| **命令调度** | 通过 `Action` 下发 GOTO / JOG / 点位 / 轨迹等命令 | 各 `*ProcessModule.Action` |
| **参数读写** | 读写速度、坐标、预设点数量等 | `SetParam` / `GetParam` |
| **报警汇聚** | 统一订阅各模组报警 | `SubscribeAlarms` |
| **项目切换** | 切换项目路径并广播重载参数 | `ProcessModuleEnvironment.SwitchProject` |
| **运动控制抽象** | 标准指令 + 真实位置反馈，换硬件不改模组 | `IMotionService` |

### 2.1 内置工艺模组

| 注册名 | 类 | 功能概要 |
|--------|-----|----------|
| `"MainControl"` | `MainControlProcessModule` | 手动操控：滑块/JOG/急停、GOTO/ORIGIN/CENTER |
| `"PointJump"` | `PointJumpProcessModule` | 预设点收藏与跳转：GOTOPOINT/SAVEPOINT/DELETEPOINT |
| `"TrajectoryView"` | `TrajectoryViewProcessModule` | 轨迹记录与显示：CLEARTRAIL/SHOWTRAIL/RANDOM |

> 注册名**不区分大小写**（`Get("maincontrol")` 与 `Get("MainControl")` 等价）。

### 2.2 标准启动流程（六步）

```
① Open 硬件服务 (IMotionService.Open)
② InitAll 初始化模组
③ InjectServiceToAll 注入服务 ★
④ SubscribeAlarms 订阅报警
⑤ ShowRunForm 显示界面
⑥ SwitchProject 打开项目（可选）
```

关闭顺序（倒序）：

```
UnsubscribeAlarms → SaveAll → CloseAll → CloseService
```

## 3. 系统架构与数据流

```
┌─────────────────────────────────────────────────────────────┐
│  上位机平台（主程序）                                          │
│  ProcessModuleManager / ProcessModuleEnvironment / 报警 UI   │
└────────────────────────────┬────────────────────────────────┘
                             │ InitAll / Inject / Action / Show*
┌────────────────────────────▼────────────────────────────────┐
│  工艺模组层（MainControl / PointJump / TrajectoryView）        │
│  ProcessModuleBase 契约：Init / Action / SetParam / Show*    │
└────────────────────────────┬────────────────────────────────┘
                             │ Hub.SetTarget / SpeedSetting / ...
┌────────────────────────────▼────────────────────────────────┐
│  业务大脑 XyzControllerHub                                    │
│  打包 MotionCommand → ExecuteCommand；订阅 PositionUpdated   │
└────────────────────────────┬────────────────────────────────┘
                             │ IMotionService（插座标准）
┌────────────────────────────▼────────────────────────────────┐
│  后端服务（PlatformMotionService 或自定义实现）                 │
│  翻译为控制卡/PLC SDK；RaisePositionUpdated 推真实位置         │
└─────────────────────────────────────────────────────────────┘
```

**两条数据流**

| 方向 | 路径 |
|------|------|
| **指令下发** | UI / Action → Hub → `MotionCommand` → `IMotionService.ExecuteCommand` → 硬件 |
| **位置反馈** | 硬件 → `PositionUpdated` / `GetPosition` → Hub 更新 `Current` → `Hub.Changed` → UI |

> ⚠️ **铁律**：`AxisPosition.Actual` 必须来自真实硬件反馈，严禁生产环境伪造位置。

## 4. 平台生命周期接口（ProcessModuleManager）

**文件**：`ProcessModules\ProcessModuleManager.cs`  
**类型**：`public static class ProcessModuleManager`  
**命名空间**：`ProcessModules`

### 4.1 API 一览

| 方法 / 属性 | 签名 | 返回 | 说明 |
|-------------|------|------|------|
| `Modules` | `IEnumerable<KeyValuePair<string, ProcessModuleBase>>` | 只读枚举 | 全部已注册模组 |
| `Get` | `ProcessModuleBase Get(string name)` | 模组或 `null` | 按注册名获取 |
| `InitAll` | `bool InitAll()` | 是否全部成功 | 初始化三个内置模组（幂等） |
| `RegisterAndInit` | `bool RegisterAndInit(ProcessModuleBase module, string name)` | 是否成功 | 注册并初始化自定义模组 |
| `InjectServiceToAll` | `void InjectServiceToAll(IMotionService service)` | — | 向全部模组注入运动服务 ★ |
| `SubscribeAlarms` | `void SubscribeAlarms(EventHandler<ModuleAlarmEventArgs> handler)` | — | 统一订阅报警 |
| `UnsubscribeAlarms` | `void UnsubscribeAlarms(EventHandler<ModuleAlarmEventArgs> handler)` | — | 取消报警订阅 |
| `StopAll` | `bool StopAll()` | 是否全部成功 | 停止所有模组运动/流程 |
| `SaveAll` | `bool SaveAll()` | 是否全部成功 | 保存所有模组参数 |
| `CloseAll` | `bool CloseAll()` | 是否全部成功 | 关闭所有模组并重置初始化标志 |

### 4.2 请求 / 响应说明

#### `InitAll()`

| 项 | 说明 |
|----|------|
| **请求参数** | 无 |
| **响应** | `true`：三个模组均 `Init` 成功；`false`：至少一个失败 |
| **副作用** | 注册 `"MainControl"` / `"PointJump"` / `"TrajectoryView"`；加载全局/项目参数；创建各模组内部 Hub |
| **幂等** | 已初始化时直接返回 `true`，不重复创建 |

```csharp
bool ok = ProcessModuleManager.InitAll();
// 响应：ok == true 表示可继续 Inject / Show
```

#### `Get(string name)`

| 项 | 说明 |
|----|------|
| **请求参数** | `name`：注册名，如 `"MainControl"`（不区分大小写） |
| **响应** | 对应 `ProcessModuleBase` 实例；未找到返回 `null` |

```csharp
ProcessModuleBase main = ProcessModuleManager.Get("MainControl");
if (main == null || !main.bInitOK)
{
    // 未初始化或初始化失败
}
```

#### `InjectServiceToAll(IMotionService service)`

| 项 | 说明 |
|----|------|
| **请求参数** | `service`：已 `Open()` 成功的运动服务实例（可为 `null`，表示断开） |
| **响应** | 无返回值 |
| **副作用** | 对每个模组调用 `SetMotionService` → Hub `SetService`：退订旧服务、订阅新服务、立即 `GetPosition` 同步 |

```csharp
IMotionService svc = new PlatformMotionService();
if (!svc.Open()) { /* 连接失败 */ return; }
ProcessModuleManager.InjectServiceToAll(svc);
// 此后各模组 Action/Hub 指令可到达硬件
```

#### `SubscribeAlarms` / `UnsubscribeAlarms`

| 项 | 说明 |
|----|------|
| **请求参数** | `handler`：`void Handler(object sender, ModuleAlarmEventArgs e)` |
| **响应** | 无 |
| **事件数据** | 见 [第 13 节](#13-报警事件-modulealarmeventargs) |

```csharp
ProcessModuleManager.SubscribeAlarms(OnModuleAlarm);

private void OnModuleAlarm(object sender, ModuleAlarmEventArgs e)
{
    // e.ModuleName, e.Message, e.Time
    Console.WriteLine("[{0}] {1} @ {2}", e.ModuleName, e.Message, e.Time);
}
```

#### `StopAll` / `SaveAll` / `CloseAll`

| 方法 | 请求 | 响应 | 说明 |
|------|------|------|------|
| `StopAll()` | 无 | `bool` 全部成功 | 调用各模组 `StopAll()` |
| `SaveAll()` | 无 | `bool` 全部成功 | 调用各模组 `Save()` |
| `CloseAll()` | 无 | `bool` 全部成功 | 调用各模组 `Close()`，并清除 `_initialized` |

### 4.3 调用示例：遍历模组

```csharp
foreach (KeyValuePair<string, ProcessModuleBase> kv in ProcessModuleManager.Modules)
{
    string name = kv.Key;
    ProcessModuleBase mod = kv.Value;
    Console.WriteLine("{0}: InitOK={1}, Info={2}", name, mod.bInitOK, mod.GetInfo());
}
```

## 7. 运动控制接口（IMotionService）

**文件**：`ProcessModules\Logic\IMotionService.cs`  
**命名空间**：`ProcessModules`

### 7.1 接口定义

```csharp
public interface IMotionService
{
    // 指令下发（前端 → 后端，唯一入口）
    bool ExecuteCommand(MotionCommand command);

    // 位置反馈（后端 → 前端）
    event EventHandler<AxisPositionEventArgs> PositionUpdated;
    AxisPosition GetPosition();

    // 状态
    bool IsConnected { get; }
    int AxisCount { get; }

    // 生命周期
    bool Open();
    void CloseService();
}
```

### 7.2 成员详表

| 成员 | 方向 | 请求参数 | 响应 / 数据 | 说明 |
|------|------|----------|-------------|------|
| `ExecuteCommand` | 下发 | `MotionCommand command` | `bool`：`true`=接受，`false`=拒绝 | 移动/回原/急停/设速等统一入口 |
| `PositionUpdated` | 推送 | — | `AxisPositionEventArgs` | 硬件位置变化时主动通知 |
| `GetPosition` | 查询 | 无 | `AxisPosition` | 初始化/重连时主动拉一次位置 |
| `IsConnected` | 状态 | — | `bool` | 通信是否正常 |
| `AxisCount` | 状态 | — | `int` | 轴数，当前为 **4**（X/Y/Z/U） |
| `Open` | 生命周期 | 无 | `bool`：连接是否成功 | 初始化控制卡/PLC |
| `CloseService` | 生命周期 | 无 | 无 | 释放资源 |

### 7.3 ExecuteCommand 请求 / 响应

**请求结构**：见 [第 8 节 MotionCommand](#8-统一指令motioncommand)

**响应**

| 返回值 | 含义 |
|--------|------|
| `true` | 后端已接受指令（≠ 运动完成） |
| `false` | 拒绝：未连接、`command==null`、参数非法、硬件拒绝等 |

**调用示例**

```csharp
MotionCommand cmd = MotionCommand.CreateMoveAbsolute(
    new float[] { 10f, 20f, 5f, 0f }, speed: 50f);
bool accepted = motionService.ExecuteCommand(cmd);
// accepted == true 仅表示指令被接受；到位需看 PositionUpdated / IsMoving
```

### 7.4 位置推送事件数据

```csharp
// 订阅
motionService.PositionUpdated += OnPositionUpdated;

void OnPositionUpdated(object sender, AxisPositionEventArgs e)
{
    AxisPosition p = e.Position;
    // p.Actual[0..3] = X/Y/Z/U 真实位置
    // p.Target[0..3] = 目标位置（若后端提供）
    // p.IsMoving / IsHomed / IsAlarmed
    // p.Timestamp
}
```

### 7.5 骨架实现

**类**：`PlatformMotionService`（`Logic\PlatformMotionService.cs`）

| 方法 | 当前行为 | 平台需完成 |
|------|----------|------------|
| `Open()` | 置 `_isConnected=true`，创建空快照 | 真实初始化控制卡 |
| `ExecuteCommand` | 校验连接后返回 `true` | 按 `CommandType` 调 SDK |
| `GetPosition` | 返回 `_lastPosition` | 主动读硬件位置 |
| `RaisePositionUpdated` | 缓存并触发事件 | 在硬件回调中调用 |
| `CloseService` | `_isConnected=false` | 释放 SDK 资源 |

```csharp
// 硬件回调中推送真实位置
var pos = AxisPosition.CreateEmpty(4);
pos.Actual = new float[] { x, y, z, u };  // 来自编码器
pos.Timestamp = DateTime.Now;
((PlatformMotionService)svc).RaisePositionUpdated(pos);
```

## 8. 统一指令（MotionCommand）

**文件**：`ProcessModules\Logic\MotionCommand.cs`

### 8.1 指令类型枚举 `MotionCommandType`

| 枚举值 | 含义 | 必填字段 | 可选字段 |
|--------|------|----------|----------|
| `MoveAbsolute` | 绝对移动到目标坐标 | `Positions` | `Speed`（0=用当前速度） |
| `MoveRelative` | 相对当前位置偏移 | `Positions`（偏移量） | `Speed` |
| `Home` | 回原点 | — | — |
| `Stop` | 减速停止 | — | — |
| `EmergencyStop` | 急停 | — | — |
| `SetSpeed` | 设置速度 | `Speed` | — |
| `SetCoordinateSystem` | 设置坐标系 | `CoordinateSystem` | — |
| `Enable` | 使能（上电） | — | — |
| `Disable` | 去使能（断电） | — | — |
| `Reset` | 复位报警 | — | — |

### 8.2 请求数据结构

```csharp
public class MotionCommand
{
    public MotionCommandType CommandType;  // 指令类型
    public float[] Positions;              // [X, Y, Z, U]，索引 0/1/2/3
    public float Speed;                    // 速度（mm/s 或档位，由后端定义）
    public int CoordinateSystem;           // 0=机械坐标，1+=工件坐标
    public object Tag;                     // 扩展（轴掩码、加速度等）
}
```

**轴索引约定**

| 索引 | 轴 |
|------|-----|
| 0 | X |
| 1 | Y |
| 2 | Z |
| 3 | U |

### 8.3 快捷构造（推荐）

| 工厂方法 | 参数 | 生成类型 |
|----------|------|----------|
| `CreateMoveAbsolute(float[] positions, float speed = 0f)` | 绝对坐标 + 速度 | `MoveAbsolute` |
| `CreateMoveRelative(float[] offsets, float speed = 0f)` | 偏移量 + 速度 | `MoveRelative` |
| `CreateSetSpeed(float speed)` | 速度值 | `SetSpeed` |
| `CreateSetCoordinateSystem(int csIndex)` | 坐标系编号 | `SetCoordinateSystem` |

### 8.4 请求示例

```csharp
// 绝对移动到 (10, 20, 5, 0)，速度 50
MotionCommand move = MotionCommand.CreateMoveAbsolute(
    new float[] { 10f, 20f, 5f, 0f }, 50f);

// 相对：X +1
MotionCommand rel = MotionCommand.CreateMoveRelative(
    new float[] { 1f, 0f, 0f, 0f });

// 设速度 80
MotionCommand spd = MotionCommand.CreateSetSpeed(80f);

// 工件坐标系 1
MotionCommand cs = MotionCommand.CreateSetCoordinateSystem(1);

// 急停（仅类型）
MotionCommand estop = new MotionCommand(MotionCommandType.EmergencyStop);
```

### 8.5 响应

经 `IMotionService.ExecuteCommand` 返回 `bool`，见 [7.3](#73-executecommand-请求--响应)。

## 9. 位置快照（AxisPosition）

**文件**：`ProcessModules\Logic\AxisPosition.cs`

### 9.1 响应数据结构

```csharp
public class AxisPosition
{
    public float[] Target;       // 目标位置 [X,Y,Z,U]
    public float[] Actual;       // 实际位置（★必须来自硬件）
    public bool[]  IsMoving;     // 是否运动中
    public bool[]  IsHomed;      // 是否已回原
    public bool[]  IsAlarmed;    // 是否报警
    public DateTime Timestamp;   // 时间戳

    public int AxisCount { get; }                    // Actual.Length
```

**紧急停止行为**

1. 调用 `_hub.EmergencyStop()` → 向硬件发送急停指令
2. `MachineStatus.bSemiProcess = false`（清除半自动标志）
3. 各 JOG 服务执行物理急停（刹车/断电）

> ⚠️ **平台责任**：建议在平台 UI 上提供独立的「急停按钮」,直接触发各模组的 ESTOP，优先级高于所有其他操作！

```csharp
        return true;

        ProcessModuleManager.SaveAll();
        ProcessModuleManager.CloseAll();
        if (_motionService != null)
            _motionService.CloseService();
    }

    private void OnModuleAlarm(object sender, ModuleAlarmEventArgs e)
    {
```

**③** `IMotionService` 实现的 `ExecuteCommand` 增加 `case`

```csharp
// Hub
public void SetAcceleration(float acc)
{
    MotionCommand cmd = new MotionCommand(MotionCommandType.SetAcceleration);
    cmd.Speed = acc; // 或扩展 MotionCommand 字段 / Tag
    SendCommand(cmd);
}
```

## 17. 错误码与约定

### 17.1 Action 返回码

| 码 | 含义 | 典型场景 |
|----|------|----------|
| `0` | 成功 | 命令已执行/已下发 |
| `-1` | 未知或空命令 | `param` 空；命令名不在 switch |
| `-2` | 参数/业务错误 | 缺参、格式错、预设不存在、轴名非法 |

### 17.2 ExecuteCommand 返回值

| 值 | 含义 |
|----|------|
| `true` | 后端接受指令 |
| `false` | 未连接 / null / 非法 / 硬件拒绝 |

### 17.3 布尔生命周期返回

`Init` / `InitAll` / `Save` / `StopAll` / `Close` / `Open` 等：`true`=成功，`false`=失败。

### 17.4 线程约定

| 场景 | 约定 |
|------|------|
| 硬件位置回调 | 可能在非 UI 线程 |
| UI 刷新 | 需 `Control.Invoke` / `BeginInvoke`（RunForm 内已处理） |
| 工控机编译 | 避免 Lambda；事件用命名方法 + 显式委托 |

## 18. FAQ

**Q1：界面显示但机器不动？**  
漏调 `InjectServiceToAll`，或 `Open()` 失败（`IsConnected==false`）。

**Q2：位置不刷新？**  
后端未触发 `PositionUpdated`，或 `Actual` 未写入真实值。

**Q3：收不到报警？**  
未 `SubscribeAlarms`，或 handler 签名不匹配。

**Q4：`Get` 返回 null？**  
未先 `InitAll`，或注册名拼写错误。

**Q5：`ExecuteCommand` 返回 true 但轴未到位？**  
`true` 只表示接受指令；到位看 `IsMoving`/`Actual` 反馈。

**Q6：没有硬件如何测界面？**  
实现回环服务：在 `ExecuteCommand` 后用目标坐标 `RaisePositionUpdated`；仅限调试。

**Q7：编译找不到类型？**  
检查 `using InterfaceDefine` / `MainModule` / `ProcessModules`，以及 DLL 引用。

**Q8：能否中途换硬件？**  
可以，重新 `InjectServiceToAll(新服务)` 即可。
