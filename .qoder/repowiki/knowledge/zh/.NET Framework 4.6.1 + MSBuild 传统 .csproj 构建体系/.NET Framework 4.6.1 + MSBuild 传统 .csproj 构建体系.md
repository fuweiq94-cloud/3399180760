## 构建系统概览

本项目采用传统的 .NET Framework 4.6.1 + Visual Studio 2017+ 生态，基于 **MSBuild** 和老式（非 SDK 风格）`.csproj` 文件进行编译、链接与产物组织。整个解决方案由 `XyzController.sln` 统一编排四个项目，无外部 CI/Makefile/Docker 等跨平台构建脚本。

## 核心构建工件

- **解决方案文件**：`XyzController.sln`，声明 Debug|Any CPU / Release|Any CPU 两种配置，所有项目均绑定到这两个配置。
- **四个项目**：
  - `src/XyzController/XyzController.csproj` — WinForms 业务库（OutputType=Library），引用 Controls 与 WpfHost。
  - `src/XyzController.Controls/XyzController.Controls.csproj` — 自定义控件类库，输出 DLL。
  - `src/XyzController.WpfHost/XyzController.WpfHost.csproj` — WPF 宿主库 + 后置生成独立 EXE 启动器。
  - `src/XyzController.Tests/XyzController.Tests.csproj` — 自托管控制台测试程序（OutputType=Exe），不依赖 NUnit/xUnit。

## 关键构建约定与决策

### 1. 老式 .csproj + 显式 Import
所有 `.csproj` 均为 ToolsVersion="14.0" 的非 SDK 风格，末尾必须显式 `<Import Project="$(MSBuildToolsPath)\Microsoft.CSharp.targets" />` 才能触发 C# 编译；WPF 宿主还额外导入 WPF 目标。

### 2. 双输出策略（DLL + EXE）
WpfHost 项目通过自定义 MSBuild Target `BuildExe`（AfterTargets="Build"）调用内联 `Csc` 任务，将 `Bootstrapper.cs` 单独编译为 `XyzController.WpfHost.Launcher.exe`。注释中明确记录了避免 CLR 把 EXE 自身当作同名程序集导致 TypeLoadException 的设计考量。

### 3. 调试输出路径特殊处理
Debug 模式下，主项目和控件库的 OutputPath 被重定向到 `..\XyzController.WpfHost\bin\Debug\`，使 VS 调试时能直接加载最新 DLL，无需手动复制。

### 4. 版本与语言特性
- 目标框架固定为 `v4.6.1`，所有项目一致。
- `LangVersion=latest` 启用最新 C# 语法，但 WarningLevel 在控件库中被限制为 4（VS2017 编译器上限），否则报 CS1900。
- `Deterministic=true` 开启确定性构建。

### 5. 依赖方向与循环约束
- 业务层单向依赖：`XyzController → XyzController.Controls + XyzController.WpfHost`。
- WpfHost 不反向引用业务项目，仅暴露 `WpfHostLauncher.Run(Form)` 接口，避免循环依赖。
- Tests 仅引用主项目，不依赖任何第三方测试框架。

### 6. 忽略规则
`.gitignore` 排除了 `*.suo`、`*.user`、`obj/`、`bin/` 等 VS 本地缓存与中间产物，符合 .NET 常规实践。

## 开发者应遵循的规则

1. **新增项目时必须同步更新 `XyzController.sln`**，并为其分配新的 ProjectGuid。
2. **保持 TargetFrameworkVersion=v4.6.1 不变**，除非整体迁移到 .NET Framework 更高版本或 .NET Core/6+。
3. **不要引入 SDK 风格 .csproj**，否则会破坏现有 MSBuild 目标链路与 WpfHost 的 `BuildExe` 后置目标。
4. **如需生成可执行入口，复用 WpfHost 的 `BuildExe` 模式**：通过自定义 Target + 内联 `Csc` 任务，确保 EXE 文件名与 DLL 不同。
5. **调试输出路径重定向仅在 Debug 配置下使用**，Release 仍走默认 bin/Release 目录。
6. **WarningLevel 不得超过 4**，以兼容 VS2017 内置编译器。
