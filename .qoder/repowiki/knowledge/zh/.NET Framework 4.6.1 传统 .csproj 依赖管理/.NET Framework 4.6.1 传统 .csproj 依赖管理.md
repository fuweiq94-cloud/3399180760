## 依赖管理系统概述

本项目采用传统的 .NET Framework 4.6.1 + Visual Studio 2017（MSBuild ToolsVersion 14.0）构建体系，未使用 NuGet、包管理器或任何第三方依赖。所有依赖均为 .NET Framework 框架程序集，通过 Reference 元素在 csproj 中显式声明。

## 关键文件与结构

- 解决方案文件: XyzController.sln — 定义三个项目及其配置映射
- 主应用项目: src/XyzController/XyzController.csproj — WinForms 可执行程序
- 控件库项目: src/XyzController.Controls/XyzController.Controls.csproj — 自定义 UI 控件类库
- 测试项目: src/XyzController.Tests/XyzController.Tests.csproj — 自托管测试运行器

## 架构与约定

### 1. 依赖声明方式
所有依赖均通过 ItemGroup/Reference 显式引用：
- 框架程序集：System、System.Drawing、System.Windows.Forms、Microsoft.CSharp、System.Core
- 项目间引用：通过 ProjectReference 引用同解决方案内的其他项目

### 2. 无第三方依赖策略
- 零外部包：不使用 NuGet、没有 packages.config、没有 nuget.config
- 纯框架依赖：仅依赖 .NET Framework 4.6.1 提供的系统程序集
- 自实现测试框架：测试项目不依赖 NUnit/xUnit，而是自行实现 Assert、TestAttribute、TestRunner

### 3. 构建配置
- 使用传统非 SDK 风格的 csproj 格式
- 必须手动导入 Microsoft.CSharp.targets 才能编译
- 每个项目都定义了独立的 Debug/Release 配置和输出路径

## 开发者应遵循的规则

1. 新增依赖时：直接在对应项目的 ItemGroup 中添加 Reference 元素
2. 添加新项目：在 sln 文件中注册项目引用关系
3. 避免引入第三方包：保持零外部依赖的简洁性
4. 版本锁定：通过固定 TargetFrameworkVersion=v4.6.1 确保运行时一致性
