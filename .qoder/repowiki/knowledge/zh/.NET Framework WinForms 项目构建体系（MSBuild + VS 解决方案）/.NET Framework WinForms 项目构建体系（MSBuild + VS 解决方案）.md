## 构建系统概览

本项目采用传统的 .NET Framework 4.6.1 + MSBuild 构建体系，通过 Visual Studio 2022（Solution File Format 12.00）管理三个子项目的编译与依赖关系。所有项目均使用老式非 SDK 风格的 `.csproj` 文件，显式 `Import Microsoft.CSharp.targets` 以启用 C# 编译目标。

## 核心构建工件

- **解决方案**：`XyzController.sln`，定义全局配置（Debug/Release × AnyCPU）和项目间依赖
- **主应用**：`src/XyzController/XyzController.csproj` → 输出 `WinExe`（`XyzController.exe`），引用控件库
- **控件库**：`src/XyzController.Controls/XyzController.Controls.csproj` → 输出 `Library`（`XyzController.Controls.dll`），提供 XYView/ZBarView/JogButton 等自定义控件
- **测试程序**：`src/XyzController.Tests/XyzController.Tests.csproj` → 输出 `Exe`（`XyzController.Tests.exe`），自托管测试运行器，不依赖 NUnit/xUnit

## 构建配置约定

| 维度 | Debug | Release |
|------|-------|---------|
| 优化 | `Optimize=false` | `Optimize=true` |
| 调试符号 | `full` | `pdbonly` |
| 条件常量 | `DEBUG;TRACE` | `TRACE` |
| 警告级别 | 2（宽松） | 默认（严格） |
| 确定性构建 | `Deterministic=true` | `Deterministic=true` |
| 语言版本 | `latest` | `latest` |

所有项目统一目标框架 `v4.6.1`、平台 `AnyCPU`，并通过 `<ProjectReference>` 声明项目级依赖而非二进制引用，确保增量编译正确性。

## 目录结构与输出约定

- 每个项目遵循标准 `bin\Debug\` / `bin\Release\` 输出路径
- 资源文件（如 `MainForm.resx`）通过 `<EmbeddedResource>` 嵌入到程序集
- 设计器生成的代码（`.Designer.cs`）通过 `<DependentUpon>` 与逻辑文件关联
- 自定义控件标记为 `<SubType>Component</SubType>`，以便在 VS 工具箱中注册

## 开发者工作流

1. 在 Visual Studio 2022 中打开 `XyzController.sln` 进行开发
2. 选择 Debug/Release 和 AnyCPU 配置后直接 F5/F6 构建
3. 测试通过双击 `bin\Debug\XyzController.Tests.exe` 或从 VS 启动
4. 无外部 CI/CD、Dockerfile、Makefile 或 NuGet 包管理——纯本地手动构建

## 约束与建议

- 新增类文件需同步添加到对应 `.csproj` 的 `<Compile Include="..." />` 列表
- 新增第三方依赖需手动添加 `<Reference Include="..." />` 条目
- 如需跨平台或容器化部署，当前体系需迁移至 .NET 8+ SDK 风格项目并引入 Dockerfile/CI 流水线
